- sqlite3 CLI conventions (default mode, no flags): rows are printed one per line; columns within a
  row are joined by `|`; no header row, no borders, no row count summary. A single-column query
  (e.g. `SELECT DISTINCT x ...`, `SELECT ... UNION SELECT ...`) yields newline-separated values with
  no separators. A NULL value prints as an empty string (rendered as a bare line) unless the query
  coerces it. Empty result sets print nothing (empty stdout). Successful writes/DDL/PRAGMAs print
  nothing. `.schema` prints CREATE statements in creation order, each terminated with `;`, blank
  line between is not added. Errors print to stderr as `Error: <message>` and are marked as errors.
